# دليل تحديث تطبيق نقطة البيع (Remote Updates Guide) 🚀

يمتلك تطبيق LibraryOS POS نظام تحديث تلقائي (Auto-Updater) يسمح لك بإرسال تحديثات لجميع أجهزة الكاشير عن بعد بضغطة زر واحدة.

## الخطوات الأساسية لإصدار تحديث جديد:

### 1. تجهيز الكود للبناء (Local Build)
تأكد من تحديث رقم الإصدار (Version) في ملف `package.json` داخل مجلد `artifacts/pos-electron/package.json`.
مثال: من `1.0.0` إلى `1.0.1`.

بعدها قم بتشغيل أمر البناء:
```bash
pnpm --filter @workspace/pos-electron run package:linux
```

### 2. الملفات المطلوبة للرفع
بعد البناء، ستجد ملفات جديدة في مجلد `dist`:
- ملف التطبيق: `libraryos-pos-1.0.1-x86_64.AppImage` (أو .deb).
- ملف التعريف: `latest-linux.yml` (هذا الملف ضروري ليعرف التطبيق بوجود تحديث).

### 3. الرفع على GitHub (The Deploy)
1. اذهب إلى مستودع الإصدارات الخاص بك على GitHub (مثل `libraryos/pos-releases`).
2. قم بإنشاء **New Release**.
3. اجعل الـ **Tag version** يطابق رقم الإصدار (مثل `v1.0.1`).
4. قم بسحب وإفلات الملفين المذكورين أعلاه (`.AppImage` و `latest-linux.yml`) إلى منطقة الرفع.
5. اضغط على **Publish release**.

### 4. كيف سيعمل التحديث لدى الكاشير؟
- بمجرد تشغيل تطبيق الـ POS لدى الكاشير، سيقوم بالتحقق من رابط GitHub.
- سيظهر شريط تحميل (Progress Bar) أزرق في الشريط العلوي للتطبيق.
- عند انتهاء التحميل، ستتغير الرسالة إلى "تثبيت التحديث الآن".
- بمجرد الضغط عليها، سيغلق التطبيق ويفتح بالنسخة الجديدة فوراً.

> [!TIP]
> يمكنك تخصيص رابط الرفع في ملف `electron/updater.ts` إذا قمت بتغيير اسم المستودع.
